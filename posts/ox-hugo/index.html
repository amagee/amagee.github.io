ox-hugo

Jan 13, 2021
1 min read

Compiles Org Mode files to Hugo markdown files, so essentially duplicates a lot of the functionality of Hugo itself, but has the advantage that you don’t need any Hugo-specific syntax in your Org files to make them work.

To setup, add

dotspacemacs-configuration-layers
'(
  ;; ...
  (org :variables
       org-enable-hugo-support t)
  ;; ...
  )

to spacemacs config. Sources: this Github comment, Shane Sveller.

Then if you put the following in your file:

#+HUGO_SECTION: posts
#+HUGO_BASE_DIR: ~/docs/web

Then C-C C-E H H will write the file converted to markdown to ~/docs/web/content/posts/ox-hugo.md.

Why “content”?

ok-hugo docs: By default, the Markdown files reside in a hierarchy under the content/ directory in the site root directory (ref)”

LaTeX support

It doesn’t work out of the box; you need to include a MathJAX config and the scripts in your Hugo theme templates. Reference

Setting up autosave

Add the following to the bottom of your file:

* Footnotes
* COMMENT Local Variables                                           :ARCHIVE:
# Local Variables:
# eval: (org-hugo-auto-export-mode)
# End:

Note: it won’t write the Markdown if you haven’t changed the file since you last saved.

Note: sometimes it still just doesn’t work, but closing & reopening Emacs seems to fix.
